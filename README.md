1	# CMR Renamer
     2	
     3	## Быстрый запуск
     4	
     5	1. Подготовьте папку с исходными PDF, например `input/`.
     6	2. Подготовьте `manifest.json` (массив объектов `DocumentMeta`).
     7	3. Запустите:
     8	
     9	```bash
    10	python cmr_renamer.py \
    11	  --input-dir ./input \
    12	  --output-dir ./output \
    13	  --manifest ./manifest.json
    14	```
    15	
    16	После выполнения:
    17	- переименованные файлы будут в `output/`
    18	- отчёт будет в `output/report.json`
    19	
    20	## Проверка ZIP перед обработкой
    21	
    22	Если получен архив, можно включить контроль количества PDF в ZIP против количества записей в `manifest.json`:
    23	
    24	```bash
    25	python cmr_renamer.py \
    26	  --input-dir ./input \
    27	  --output-dir ./output \
    28	  --manifest ./manifest.json \
    29	  --zip ./docs.zip
    30	```
    31	
    32	При несовпадении количества программа завершится с ошибкой.
    33	
    34	## Пример `manifest.json`
    35	
    36	```json
    37	[
    38	  {
    39	    "original_name": "00000001.PDF",
    40	    "colt_numbers": ["1040378"],
    41	    "sender": "FUTURUS FOOD SIA",
    42	    "consignee": "MAXIMA EESTI OU",
    43	    "doc_date": "30.04.2026",
    44	    "rotate_180": true,
    45	    "zone24_signed": false
    46	  },
    47	  {
    48	    "original_name": "00000002.PDF",
    49	    "sender": "AD BALTIC",
    50	    "place_of_delivery": "Ringtee 35 Tartu",
    51	    "doc_date": "28.04.2026"
    52	  }
    53	]
    54	```
    55	
    56	## Алгоритм обработки CMR-документов
    57	
    58	### 1. Определить тип документа
    59	Для каждого PDF сначала определить тип:
    60	- CMR
    61	- Manifest / delivery list
    62	- Invoice
    63	- Другой документ
    64	
    65	Правила:
    66	- CMR обрабатывается по правилам переименования.
    67	- Manifest / delivery list обычно не переименовывается по CMR-правилам.
    68	- Если документ не подходит под правила, его имя оставляется без изменений.
    69	
    70	### 2. Проверить ориентацию CMR
    71	Перед финальной обработкой выполнить визуальную проверку ориентации:
    72	- если CMR перевернут вверх ногами — повернуть на `180°`
    73	- CMR переворачиваем
    74	- Manifest / delivery list не переворачиваем, если отдельно не указано
    75	
    76	### 3. Найти COLT / COPL
    77	В каждом CMR в первую очередь ищем номера:
    78	- COLT
    79	- COPL
    80	
    81	Если найден хотя бы один номер COLT/COPL, имя файла формируется по ним.
    82	
    83	**Приоритетное правило:** `COLT / COPL` всегда выше остальных правил именования.
    84	
    85	Пример:
    86	- `COLT1040378.pdf`
    87	
    88	### 4. COPL не менять на COLT
    89	Если найден `COPL`, его нельзя переименовывать в `COLT`.
    90	
    91	Правильно:
    92	- `COPL1234567.pdf`
    93	
    94	Неправильно:
    95	- `COLT1234567.pdf`
    96	
    97	Если в одном документе есть и COLT, и COPL — указывать оба:
    98	- `COLT1040001 COPL1234567.pdf`
    99	
   100	### 5. Несколько COLT / COPL в одном документе
   101	Если в одном CMR несколько номеров, включать все номера в имя через пробел.
   102	
   103	Пример:
   104	- `COLT1040133 COLT1040134 COLT1040136 COLT1040137.pdf`
   105	
   106	Рекомендуемый порядок — как в документе.
   107	
   108	### 6. Дубли номеров
   109	Если один и тот же номер встречается в нескольких PDF, добавлять индекс дубля:
   110	- `COLT1040920.pdf`
   111	- `COLT1040920.2.pdf`
   112	- `COLT1040920.3.pdf`
   113	
   114	Если в одном имени есть повторяющиеся номера:
   115	- `COLT1234567 COLT1234567.2 COLT1234567.3.pdf`
   116	
   117	Если это страницы одного CMR (например, 1/2 и 2/2) и номер одинаковый, вторая страница получает суффикс `.2`.
   118	
   119	### 7. Если COLT / COPL нет
   120	Если COLT/COPL отсутствуют, использовать структуру:
   121	
   122	`[Sender + Consignee + address + date].pdf`
   123	
   124	Это правило применяется ко всем документам без COLT/COPL, если данные можно определить надежно.
   125	
   126	Источники данных:
   127	- `Sender` — отправитель
   128	- `Consignee` — получатель
   129	- `Address` — адрес получателя / место доставки
   130	- `Date` — дата CMR, invoice или основного документа
   131	
   132	Примеры:
   133	- `AGCO AGRILAND OUE TUEKI 3 EE-61715 TORVANDI TARTUMAA 30.04.2026.pdf`
   134	- `FUTURUS MAXIMA EESTI OU TARTU MNT 49 TALLINN 30.04.2026.pdf`
   135	
   136	При дубле имени:
   137	- `FUTURUS MAXIMA EESTI OU TARTU MNT 49 TALLINN 30.04.2026.pdf`
   138	- `FUTURUS MAXIMA EESTI OU TARTU MNT 49 TALLINN 30.04.2026.2.pdf`
   139	
   140	### 8. Если данных для переименования нет
   141	Если нет COLT/COPL и невозможно надежно определить хотя бы одно из полей:
   142	- Sender
   143	- Consignee
   144	- Address
   145	- Date
   146	
   147	файл оставляется с исходным именем.
   148	
   149	Пример:
   150	- `00000001.PDF`
   151	
   152	### 9. Проверить зону 24
   153	Для каждого CMR проверить зону 24:
   154	- Подпись/печать есть — нормально
   155	- Подписи нет — отметить как проблему
   156	- Подпись сомнительная — отметить как спорное место
   157	- Manifest / delivery list — не считать CMR
   158	
   159	### 10. Проверить безопасность имени файла
   160	Перед сохранением очистить имя:
   161	- удалить/заменить запрещенные символы
   162	- убрать двойные пробелы
   163	- убрать переносы строк
   164	- сократить или упростить слишком длинные/нечитаемые фрагменты
   165	
   166	Пример нормализации:
   167	- `Rabaküla 4/3` → `Rabaküla 4-3`
   168	
   169	### 11. Финальный контроль
   170	Перед архивированием проверить:
   171	- количество PDF на входе и выходе
   172	- отсутствие потерь файлов
   173	- корректную ориентацию CMR
   174	- визуальную проверку COLT/COPL
   175	- отсутствие ошибок в цифрах COLT/COPL
   176	- отсутствие запрещенных символов в именах
   177	- проверку зоны 24
   178	- отметку спорных документов
   179	
   180	Важно: номера COLT/COPL обязательно проверять визуально, т.к. OCR может путать:
   181	- `8 ↔ 3`
   182	- `0 ↔ O`
   183	- `1 ↔ I`
   184	- порядок цифр
   185	
   186	## Короткая схема
   187	1. Определить тип PDF.
   188	2. Проверить ориентацию CMR.
   189	3. Найти COLT / COPL.
   190	4. Если COLT / COPL есть → назвать файл по ним.
   191	5. Если COLT / COPL нет → назвать по структуре `[Sender + Consignee + address + date]`.
   192	6. Если данных недостаточно → оставить исходное имя.
   193	7. Проверить дубли и добавить `.2`, `.3`.
   194	8. Очистить имя файла.
   195	9. Проверить зону 24.
   196	10. Сверить входное и выходное количество PDF.
   197	11. Подготовить архив.
   198	
   199	## Главное правило
   200	`COLT / COPL → Sender + Consignee + address + date → исходное имя`
