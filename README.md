+## Алгоритм обработки CMR-документов
+
+### 1. Определить тип документа
+Для каждого PDF сначала определить тип:
+- CMR
+- Manifest / delivery list
+- Invoice
+- Другой документ
+
+Правила:
+- CMR обрабатывается по правилам переименования.
+- Manifest / delivery list обычно не переименовывается по CMR-правилам.
+- Если документ не подходит под правила, его имя оставляется без изменений.
+
+### 2. Проверить ориентацию CMR
+Перед финальной обработкой выполнить визуальную проверку ориентации:
+- если CMR перевернут вверх ногами — повернуть на `180°`
+- CMR переворачиваем
+- Manifest / delivery list не переворачиваем, если отдельно не указано
+
+### 3. Найти COLT / COPL
+В каждом CMR в первую очередь ищем номера:
+- COLT
+- COPL
+
+Если найден хотя бы один номер COLT/COPL, имя файла формируется по ним.
+
+**Приоритетное правило:** `COLT / COPL` всегда выше остальных правил именования.
+
+Пример:
+- `COLT1040378.pdf`
+
+### 4. COPL не менять на COLT
+Если найден `COPL`, его нельзя переименовывать в `COLT`.
+
+Правильно:
+- `COPL1234567.pdf`
+
+Неправильно:
+- `COLT1234567.pdf`
+
+Если в одном документе есть и COLT, и COPL — указывать оба:
+- `COLT1040001 COPL1234567.pdf`
+
+### 5. Несколько COLT / COPL в одном документе
+Если в одном CMR несколько номеров, включать все номера в имя через пробел.
+
+Пример:
+- `COLT1040133 COLT1040134 COLT1040136 COLT1040137.pdf`
+
+Рекомендуемый порядок — как в документе.
+
+### 6. Дубли номеров
+Если один и тот же номер встречается в нескольких PDF, добавлять индекс дубля:
+- `COLT1040920.pdf`
+- `COLT1040920.2.pdf`
+- `COLT1040920.3.pdf`
+
+Если в одном имени есть повторяющиеся номера:
+- `COLT1234567 COLT1234567.2 COLT1234567.3.pdf`
+
+Если это страницы одного CMR (например, 1/2 и 2/2) и номер одинаковый, вторая страница получает суффикс `.2`.
+
+### 7. Если COLT / COPL нет
+Если COLT/COPL отсутствуют, использовать структуру:
+
+`[Sender + Consignee + address + date].pdf`
+
+Это правило применяется ко всем документам без COLT/COPL, если данные можно определить надежно.
+
+Источники данных:
+- `Sender` — отправитель
+- `Consignee` — получатель
+- `Address` — адрес получателя / место доставки
+- `Date` — дата CMR, invoice или основного документа
+
+Примеры:
+- `AGCO AGRILAND OUE TUEKI 3 EE-61715 TORVANDI TARTUMAA 30.04.2026.pdf`
+- `FUTURUS MAXIMA EESTI OU TARTU MNT 49 TALLINN 30.04.2026.pdf`
+
+При дубле имени:
+- `FUTURUS MAXIMA EESTI OU TARTU MNT 49 TALLINN 30.04.2026.pdf`
+- `FUTURUS MAXIMA EESTI OU TARTU MNT 49 TALLINN 30.04.2026.2.pdf`
+
+### 8. Если данных для переименования нет
+Если нет COLT/COPL и невозможно надежно определить хотя бы одно из полей:
+- Sender
+- Consignee
+- Address
+- Date
+
+файл оставляется с исходным именем.
+
+Пример:
+- `00000001.PDF`
+
+### 9. Проверить зону 24
+Для каждого CMR проверить зону 24:
+- Подпись/печать есть — нормально
+- Подписи нет — отметить как проблему
+- Подпись сомнительная — отметить как спорное место
+- Manifest / delivery list — не считать CMR
+
+### 10. Проверить безопасность имени файла
+Перед сохранением очистить имя:
+- удалить/заменить запрещенные символы
+- убрать двойные пробелы
+- убрать переносы строк
+- сократить или упростить слишком длинные/нечитаемые фрагменты
+
+Пример нормализации:
+- `Rabaküla 4/3` → `Rabaküla 4-3`
+
+### 11. Финальный контроль
+Перед архивированием проверить:
+- количество PDF на входе и выходе
+- отсутствие потерь файлов
+- корректную ориентацию CMR
+- визуальную проверку COLT/COPL
+- отсутствие ошибок в цифрах COLT/COPL
+- отсутствие запрещенных символов в именах
+- проверку зоны 24
+- отметку спорных документов
+
+Важно: номера COLT/COPL обязательно проверять визуально, т.к. OCR может путать:
+- `8 ↔ 3`
+- `0 ↔ O`
+- `1 ↔ I`
+- порядок цифр
+
+## Короткая схема
+1. Определить тип PDF.
+2. Проверить ориентацию CMR.
+3. Найти COLT / COPL.
+4. Если COLT / COPL есть → назвать файл по ним.
+5. Если COLT / COPL нет → назвать по структуре `[Sender + Consignee + address + date]`.
+6. Если данных недостаточно → оставить исходное имя.
+7. Проверить дубли и добавить `.2`, `.3`.
+8. Очистить имя файла.
+9. Проверить зону 24.
+10. Сверить входное и выходное количество PDF.
+11. Подготовить архив.
+
+## Главное правило
+`COLT / COPL → Sender + Consignee + address + date → исходное имя`
